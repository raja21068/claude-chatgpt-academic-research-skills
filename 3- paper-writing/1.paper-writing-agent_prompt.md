# Paper Writing Agent — Master Execution Prompt

Copy-paste the relevant block below into Claude depending on what you want to do.
Every prompt references your actual agent structure at `paper-writing-agent/`.

---

## 1. START FROM SCRATCH (raw notes → full paper plan)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file first, then read `paper-writing-agent/references/my_writing_style.md` and
`paper-writing-agent/references/domain_vocabulary.md` if they exist.

I want to start a new paper from my research notes.

## My materials
[PASTE YOUR NOTES / IDEA / EXPERIMENT DESCRIPTION HERE]

## Target venue
[e.g. NeurIPS 2026 / CVPR / EMNLP / ICLR — or "unknown"]

## Available evidence
[Describe what you have: datasets, experiments run, results files, figures, baselines]

## What I need
Run Phase 0 (detect starting point), then Phase 1 (intake and evidence mapping),
and produce the intake summary. End with the recommended next step.

Save all outputs to `paper-writing-agent/$out/reports/`.
```

## 2. BUILD CLAIM-EVIDENCE MAP (you have results, no draft yet)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file first. Also read `paper-writing-agent/references/templates.md` for
the claim-evidence matrix format.

## My experimental results
[PASTE METRICS / TABLE / JSON SNIPPET HERE]

## My claimed contributions
[LIST 2-4 BULLET POINTS OF WHAT YOU THINK YOUR PAPER CONTRIBUTES]

Run Phase 2: build the full claim-evidence matrix for these contributions.
Before building the matrix, check whether `EXPERIMENT_AUDIT.json` exists and
read it if present.

Flag any claim as [INTEGRITY CONCERN] or [INTEGRITY WARN] if the audit
found issues. Flag unsupported claims as TODO.

Save the matrix to `paper-writing-agent/$out/reports/claim_evidence_matrix.md`.
```

## 3. WRITE A SECTION

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file first, then read:
- `paper-writing-agent/references/my_writing_style.md`  ← match my voice
- `paper-writing-agent/references/domain_vocabulary.md` ← use correct terms

## Section to write
[e.g. Introduction / Related Work / Method / Experiments / Conclusion]

## Evidence I am providing
[PASTE THE RELEVANT EXPERIMENT RESULTS, CITATIONS, OR NOTES FOR THIS SECTION]

## Outline (if you have one)
[PASTE OUTLINE HERE, or write "none — infer from evidence"]

Rules:
- Use only the evidence I provided above
- Mark missing citations as TODO:CITE
- Mark missing numbers as TODO:NUMBER
- Do not write "limitation" or "limitations" anywhere — reframe as assumptions or
  open directions
- Match my writing style from `my_writing_style.md`
- Output LaTeX [or Markdown — pick one]

Save draft to `paper-writing-agent/$out/drafts/[section_name].tex`.
```

## 4. REVIEWER SIMULATION (stress-test a draft)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file, then read `paper-writing-agent/agents/reviewer.md` before simulating.

## Paper draft
[PASTE YOUR DRAFT TEXT HERE, or reference the file path]

## Target venue
[e.g. NeurIPS / ICML / ACL]

Simulate three reviewers + area chair using the six-axis scoring framework
in `agents/reviewer.md`. Produce:
- Six-axis scores with justification and hard-cap checks
- Overall score and decision recommendation
- CRITICAL / MAJOR / MINOR weakness list with specific fixes
- Revised wording suggestions for any overclaimed statements

Save report to `paper-writing-agent/$out/reports/reviewer_simulation.md`.
```

## 5. CLAIM-NUMBER AUDIT (verify every number in the paper matches raw results)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file, then read `paper-writing-agent/agents/claim_auditor.md`.
Spawn the claim auditor as a sub-agent with ZERO prior context about this research.

Pass to the sub-agent ONLY:
- Paper source files: [LIST .tex OR .md FILE PATHS]
- Raw result files:   [LIST .json / .csv / .yaml FILE PATHS]

Do NOT pass EXPERIMENT_LOG.md, AUTO_REVIEW.md, or any summary file.

The sub-agent should produce a full claim audit table with status for every number
and a fix_instruction for every mismatch.

Save to `paper-writing-agent/$out/reports/claim_audit.md`.
After the audit, if verdict is FAIL: fix all mismatches before any further writing.
If WARN: fix listed mismatches, then continue.
```

---

## 6. EXPERIMENT INTEGRITY AUDIT (verify experiments are honestly constructed)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file, then read `paper-writing-agent/agents/experiment_auditor.md`.
Spawn the experiment auditor as a sub-agent.

Pass to the sub-agent ONLY file paths — no summaries, no interpretation:
- Evaluation scripts: [LIST eval*.py / metric*.py paths]
- Result files:       [LIST .json / .csv paths in results/ or outputs/]
- Config files:       [LIST *.yaml / args.json paths]
- Paper claims:       [LIST .tex or NARRATIVE_REPORT.md paths]

The sub-agent checks for: fake ground truth, score self-normalization,
phantom results, dead metric code, and scope overclaims.

Save report to `paper-writing-agent/$out/reports/experiment_audit.md`.
The audit is advisory — never blocks progress, but tags affected claims.
```

## 7. FINAL SUBMISSION AUDIT

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file, then read:
- `paper-writing-agent/references/checklists.md`  ← submission checklist
- `paper-writing-agent/agents/reviewer.md`         ← for final reviewer pass

## Paper to audit
[FILE PATH or PASTE FULL DRAFT]

## Venue rules
[PAGE LIMIT / ANONYMITY REQUIRED / CITATION STYLE / REQUIRED SECTIONS]

Run Phase 7 (Final Submission Audit):
1. Complete the content audit table
2. Run the claim-number audit (Phase 7b) — invoke `agents/claim_auditor.md`
3. Run venue compliance check
4. Run anonymity check (if double-blind)
5. Check that "limitation"/"limitations" does not appear in body text or headings

Final verdict must be one of:
- not ready (list blockers)
- nearly ready (list required fixes)
- ready with minor fixes (list them)
- ready

Save audit report to `paper-writing-agent/$out/reports/final_audit.md`.
Do NOT mark ready if any claim-audit verdict is FAIL or if TODO placeholders remain.
```

## 8. FULL PIPELINE (idea → submission-ready paper, one command)

```
You are operating as the paper-writing agent defined in `paper-writing-agent/SKILL.md`.
Read that file completely before doing anything else. Also read:
- `paper-writing-agent/references/my_writing_style.md`
- `paper-writing-agent/references/domain_vocabulary.md`
- `paper-writing-agent/references/checklists.md`
- `paper-writing-agent/references/templates.md`

## My research materials
[PASTE EVERYTHING: notes, results, figures description, related work, venue info]

Run the full pipeline in order:
1. Phase 0 — detect starting point
2. Phase 1 — intake and evidence mapping → save intake_summary.md
3. Phase 2 — claim-evidence matrix (check EXPERIMENT_AUDIT.json first if it exists)
4. Phase 3 — outline → save outline.md
5. Phase 4 — draft all sections → save to $out/drafts/
6. Phase 5 — citation audit → save citation_audit.md
7. Phase 6 — reviewer simulation using agents/reviewer.md → save reviewer_sim.md
8. Phase 7 — final audit including claim-number check → save final_audit.md

At each phase gate, stop and show me the output before proceeding.
Use my writing style from `my_writing_style.md` for all prose.
Never write "limitation" or "limitations" — reframe as assumptions or open directions.
Save all outputs under `paper-writing-agent/$out/` in the appropriate subfolder.
```

## Quick reference — which prompt to use

| Your situation | Use prompt |
|---|---|
| Have an idea, no draft | 1 — Start from scratch |
| Have results, need to check claims | 2 — Claim-evidence map |
| Need to write one section | 3 — Write a section |
| Have a draft, want reviewer feedback | 4 — Reviewer simulation |
| Want to verify numbers match raw data | 5 — Claim-number audit |
| Want to check experiment honesty | 6 — Experiment integrity audit |
| Ready to submit, final check | 7 — Final submission audit |
| Want everything done end-to-end | 8 — Full pipeline |
